# Automatic Client Bundle Generation

## Overview

As of version 1.1.11, Preliquify **automatically generates client-side component bundles** with **auto-registration**. This means you no longer need Vite, Rollup, or manual registration code!

## What Gets Generated

When you run `preliquify build`, you get:

1. **`.liquid` files** - Server-side rendered templates with hydration metadata
2. **`preliquify-prlq.runtime.js`** - Shared hydration runtime (one file for all components)
3. **`.bundle.js` files** - Individual component bundles with auto-registration

## How to Use

### 1. Include Scripts in Your Theme Layout

```liquid
<!-- theme.liquid -->
<!DOCTYPE html>
<html>
<head>
  <!-- ... -->
</head>
<body>
  {{ content_for_layout }}
  
  <!-- Load Preact (choose one approach) -->
  
  <!-- Option A: From CDN -->
  <script src="https://unpkg.com/preact@10.27.2/dist/preact.umd.js"></script>
  <script>window.preact = { h: preactUmd.h, render: preactUmd.render };</script>
  
  <!-- Option B: From assets (if you bundled it separately) -->
  <script src="{{ 'preact.umd.js' | asset_url }}"></script>
  <script>window.preact = { h: preactUmd.h, render: preactUmd.render };</script>
  
  <!-- Load Preliquify runtime -->
  <script src="{{ 'preliquify-prlq.runtime.js' | asset_url }}" defer></script>
  
  <!-- Load component bundles -->
  <script src="{{ 'MediaGalleryWidget-prlq.bundle.js' | asset_url }}" defer></script>
  <script src="{{ 'ProductCard-prlq.bundle.js' | asset_url }}" defer></script>
</body>
</html>
```

### 2. Render Snippets (They Auto-Hydrate!)

```liquid
<!-- product.liquid -->
<div class="product-gallery">
  {% render 'MediaGalleryWidget-prlq', media: product.media %}
</div>
```

**That's it!** The component will:
1. ✅ Render server-side (SSR) - instant display
2. ✅ Auto-register when bundle loads
3. ✅ Auto-hydrate when runtime finds it
4. ✅ Restore full interactivity

## How Auto-Registration Works

Each `.bundle.js` file contains:

```javascript
// MediaGalleryWidget-prlq.bundle.js (simplified)

import { MediaGalleryWidget } from './MediaGalleryWidget.tsx';

// Auto-registration code (generated by Preliquify)
(function() {
  let registered = false;
  
  function registerComponent() {
    if (registered) return;
    
    if (window.__PRELIQUIFY__ && window.__PRELIQUIFY__.register) {
      window.__PRELIQUIFY__.register('MediaGalleryWidget', MediaGalleryWidget);
      registered = true;
      return;
    }
    
    // Runtime not loaded yet, retry in 10ms
    setTimeout(registerComponent, 10);
  }
  
  // Start when DOM is ready
  if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', registerComponent);
  } else {
    registerComponent();
  }
})();
```

### Key Features

- ✅ **Retry logic** - Handles race conditions (bundle loads before runtime)
- ✅ **No globals** - Uses IIFE to avoid polluting window
- ✅ **DOM ready** - Works regardless of script load order
- ✅ **Automatic naming** - Uses component name from `createLiquidSnippet`

## Configuration

### Enable/Disable Client Bundles

```typescript
// preliquify.config.ts
export default {
  entryPoint: './src/snippets',
  outLiquidDir: './snippets',
  outClientDir: './assets',
  
  generateClientBundles: true,  // Default: true
  minify: true,                 // Default: true
};
```

### Disable for Testing

```typescript
export default {
  entryPoint: './src/snippets',
  generateClientBundles: false,  // Only generate .liquid files
};
```

Useful for:
- Debugging SSR output
- Testing without hydration
- CI/CD pipelines that only need templates

## Advanced: Load Order Optimization

For best performance, load scripts in this order:

```liquid
<!-- 1. Preact (dependency) -->
<script src="https://unpkg.com/preact@10.27.2/dist/preact.umd.js"></script>
<script>window.preact = { h: preactUmd.h, render: preactUmd.render };</script>

<!-- 2. Runtime (hydration framework) -->
<script src="{{ 'preliquify-prlq.runtime.js' | asset_url }}" defer></script>

<!-- 3. Component bundles (auto-register) -->
{% comment %} Load async for better performance {% endcomment %}
<script src="{{ 'MediaGalleryWidget-prlq.bundle.js' | asset_url }}" defer></script>
```

### Using `defer` and `async`

```liquid
<!-- All use defer - execute in order after DOM ready -->
<script src="{{ 'preliquify-prlq.runtime.js' | asset_url }}" defer></script>
<script src="{{ 'MediaGalleryWidget-prlq.bundle.js' | asset_url }}" defer></script>

<!-- Or async - execute as soon as downloaded (order not guaranteed) -->
{% comment %} Auto-registration handles race conditions! {% endcomment %}
<script src="{{ 'preliquify-prlq.runtime.js' | asset_url }}" async></script>
<script src="{{ 'MediaGalleryWidget-prlq.bundle.js' | asset_url }}" async></script>
```

**Both work!** The retry logic handles all load orders.

## Conditional Loading

Only load bundles for components on the page:

```liquid
<!-- Only load gallery bundle if needed -->
{% if product.media.size > 0 %}
  <script src="{{ 'MediaGalleryWidget-prlq.bundle.js' | asset_url }}" defer></script>
{% endif %}
```

## Bundle Sizes

Typical sizes (minified):

- `preliquify-prlq.runtime.js`: ~2-3 KB (shared across all pages)
- `MediaGalleryWidget-prlq.bundle.js`: ~5-15 KB (depends on component complexity)
- `preact.umd.js`: ~4 KB (from CDN)

Total hydration overhead: **~10-20 KB** for a typical page with 2-3 components.

## Troubleshooting

### Bundle Not Loading

```javascript
// Check in browser console
console.log(window.__PRELIQUIFY__);
// Should show: { register: function, hydrate: function, ... }
```

If undefined:
1. Check runtime loaded: `preliquify-prlq.runtime.js`
2. Check load order: runtime before bundles
3. Check for console errors

### Component Not Registering

```javascript
// Check registered components
window.__PRELIQUIFY__.getComponent('MediaGalleryWidget');
```

If undefined:
1. Check bundle loaded
2. Check component name matches (case-sensitive!)
3. Check `data-preliq-island` matches component name

### Hydration Not Happening

```javascript
// Check for hydration errors
window.__PRELIQUIFY__.getErrors();
```

Common issues:
- Component name mismatch
- Props parsing error
- Missing Preact
